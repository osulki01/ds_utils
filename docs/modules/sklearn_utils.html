ds_utils.sklearn_utils¶

Helper functions when working with the Scikit-learn library.

Note

The terms Scikit-learn and sklearn are used interchangeably.

Saving and Loading Scikit-learn Objects¶

Saving

When working with Scikit-learn, and machine learning in general, different versions of the library you are using can yield different outputs or inconsistent functionality.

To solve the problem of “it works on my machine”, it is generally recommended to make use of Docker, or at least a virtual environment.

However, there may still be occasions when working with Scikit-learn models that you have inherited from a colleague, or even one that you have worked on previously, when you are left wondering what version of Scikit-learn it was originally created with.

ds_utils.sk_io.save_pickled_sklearn_object_and_version() can be used to save both the Scikit-learn object, and the version of Scikit-learn currently running.

>>> from sklearn import preprocessing
>>> from ds_utils.sklearn_utils import sk_io

>>> data = [[0, 0], [0, 0], [1, 1], [1, 1]]
>>> scaler = preprocessing.StandardScaler()
>>> scaler.fit(data)

>>> sk_io.save_pickled_sklearn_object_and_version(scaler, 'scaler_and_sklearn_version.pkl')

In order to overwrite an existing file, you must request overwrite mode.

# Attempt to overwrite an existing file
>>> sk_io.save_pickled_sklearn_object_and_version(scaler, 'scaler_and_sklearn_version.pkl')
FileExistsError: File new_sklearn_file.pkl already exists.

# Successfully overwrite an existing file
>>> sk_io.save_pickled_sklearn_object_and_version(
>>>   sklearn_object=scaler,
>>>   filename_or_path='scaler_and_sklearn_version.pkl',
>>>   overwrite=True,
>>> )

Warning

Only use ds_utils.sk_io.save_pickled_sklearn_object_and_version() when saving an sklearn object you have trained/created yourself, not an already-pickled object that you have loaded from elsewhere, as you do not know whether your version of sklearn is the same as the one used to originally create it.

Loading

Your Scikit-learn object and the running version of Scikit-learn have now been saved together as a Tuple in a pickled object.

They can now be loaded using ds_utils.sk_io.save_pickled_sklearn_object_and_version(). This method will issue a warning if the version associated with the loaded object differs to the one that you are currently warning.

>>> loaded_sklearn_object, loaded_sklearn_version = sk_io.load_pickled_sklearn_object_and_version(
>>>   filename_or_path='scaler_and_sklearn_version.pkl'
>>> )

>>> print(type(loaded_sklearn_object))
<class 'sklearn.preprocessing._data.StandardScaler'>

>>> print(loaded_sklearn_version)
0.23.1

ds_utils.sklearn_utils.sk_io.save_pickled_sklearn_object_and_version(sklearn_object, filename_or_path: str, overwrite: bool = False) → None[source]¶

Saves sklearn object as a pickle file, along with the version of the sklearn library that is currently being used.

Only use when saving an sklearn object you have trained/created yourself, not an already-pickled object that you have loaded from elsewhere, as you do not know whether your version of sklearn is the same as the one used to originally create it.

Parameters

  • sklearn_object (sklearn object) – Model/sklearn-object to be saved.

  • filename_or_path (str) – Target where the object and its version will be saved.

  • overwrite (bool (default is False)) – Whether to overwrite file if it already exists.

Raises

FileExistsError – If the filename_or_path already exists and user did not set overwrite mode.
